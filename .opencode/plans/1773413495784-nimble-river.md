# Plan: Rewrite `export` command for dialog-less export

## Overview

Vexy Lines now supports dialog-less export via plist preferences. This eliminates all GUI dialog automation (pyautogui, pyperclip, mac-pyxa, multi-strategy fallbacks). The new flow: set plist prefs → launch app → open file → click menu item → poll for output file → move if needed → restore plist.

## Decisions

- **CLI arg**: Rename `target` → `input` (Fire makes it both positional and named)
- **mac-pyxa**: Remove entirely; pure AppleScript subprocess via existing `AppleScriptBridge` pattern
- **Completion detection**: Poll for output file existence + non-zero stable size
- **Tests**: Rewrite from scratch for new architecture
- **WindowWatcher**: Keep simplified (remove pattern-matching, keep `wait_for_contains`/`wait_for_any`)
- **Exporter layout**: Single `exporter.py` at package level (no `exporters/` directory needed)
- **No BaseExporter ABC**: Single concrete `VexyLinesExporter` class

### Task 2.2 — Create PlistManager (NEW)

**File:** `src/vexy_lines_utils/core/plist.py`

```python
class PlistManager:
    """Context manager for safe plist modification with backup/restore."""

    EXPORT_PREFERENCES = {
        "export·dlg·checkLayers": True,
        "export·dlg·checkMergeColor": False,
        "export·dlg·exportMode": 1,
        "export·dlg·imageFormat": 0,
        "export·dlg·radioColor": False,
        "export·dlg·radioTransparent": True,
        "export·dlg·scale": 1.0,
        "first_start": False,
    }
    # format key "export·dlg·format" set dynamically from config

    def __init__(self, plist_path: Path, format: str, app_name: str = "Vexy Lines"): ...
    def __enter__(self) -> Self:
        # 1. Quit app if running (osascript)
        # 2. Read plist with plistlib (default {} if missing)
        # 3. Store originals (use _MISSING sentinel for absent keys)
        # 4. Merge export prefs + format key
        # 5. Write plist atomically (write tmp, rename)
        return self
    def __exit__(self, *exc_info) -> None:
        # Restore originals: put back old values, delete keys that were absent
        # Always runs, even on exception
```

**Key design decisions:**
- Use `_MISSING = object()` sentinel to distinguish "key absent" from "key was False/0/empty"
- Use `plistlib.load()`/`plistlib.dump()` — handles both binary and XML plists
- Atomic write: write to temp file, then `os.replace()` to target
- If plist doesn't exist (fresh install): create it, delete on restore
- Add `PLIST_ERROR` code to `core/errors.py`

**Verification:**
- Missing plist → creates and restores (deletes) cleanly
- Existing keys → values restored exactly
- Missing original keys → removed on restore
- Exception inside `with` block → restore still occurs
- Corrupt plist → raises `AutomationError("PLIST_ERROR")` with suggestion

    def __init__(self, config: ExportConfig, *, dry_run: bool = False): ...

    def export(self, input_path: Path, output_path: Path | None = None) -> ExportStats:
        files = find_lines_files(input_path)
        # Validate output mapping upfront:
        #   dir input + file output → raise AutomationError
        
        if self.dry_run:
            # Record all files as success, return stats
            return stats

        with PlistManager(plist_path, config.format, config.app_name) as plist:
            bridge = AppleScriptBridge(config)
            # PlistManager.__enter__ already quit app and set prefs
            
            # Launch app
            subprocess.run(["open", "-a", config.app_name])
            watcher = WindowWatcher(bridge.window_titles, config.poll_interval)
            watcher.wait_for_any(config.scale_timeout(config.wait_for_app))
            
            for file in files:
                self._process_file(file, bridge, watcher, output_path, stats)
            
            # Quit app (plist restored in __exit__)
            bridge.quit_app()
        
        return stats
