The underlying Vexy Lines has changed, so now we can do the following on macOS: 

1. Close Vexy Lines if it’s running. 

2. Edit `$HOME/Library/Preferences/com.vexy-art.lines.plist` (note: this may be XML or binary plist!) so that we have: 

```xml
<key>export·dlg·checkLayers</key>
<true/>
<key>export·dlg·checkMergeColor</key>
<false/>
<key>export·dlg·exportMode</key>
<integer>1</integer>
<key>export·dlg·format</key>
<string>pdf</string>
<key>export·dlg·imageFormat</key>
<integer>0</integer>
<key>export·dlg·radioColor</key>
<false/>
<key>export·dlg·radioTransparent</key>
<true/>
<key>export·dlg·scale</key>
<real>1</real>
<key>first_start</key>
<false/>
```
(Store the previous values somewhere). 

Default --format should be 'pdf' but if I use 'svg' then 'svg' should go into "export·dlg·format"

3. Launch Vexy Lines. 

4. Open the file if --input was a file (or the first .lines file in the folder if --input was a folder)

5. Invoke ==File > Export PDF File== or ==File > Export SVG File== (see /Users/adam/Developer/vcs3/github.vexyart/vexy-lines-utils/issues/export-pdf-file.png and /Users/adam/Developer/vcs3/github.vexyart/vexy-lines-utils/issues/export-svg-file.png )

This will do a DIALOG-LESS export into the same folder and basename as the .lines file, with .pdf or .svg extension. 

6. Close the file. 

7. If I specified --output then: 
  - if --input was a file, then move the exported file into the --output file. 
  - if --input was a folder, then move the exported file into the --output folder with the same basename as the input file but with the .pdf or .svg extension.

8. Iterate over the remaining files if --input was a folder. 

9. Close the app, restore the prefs.
