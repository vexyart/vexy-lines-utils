
## SVG to PNG

https://pypi.org/project/resvg-py/

The module takes in utf-8 encoded svg_string and returns base64 encoded PNG string. 

```python
import resvg_py

svg_string = """
<svg width="300" height="130" xmlns="http://www.w3.org/2000/svg">
  <rect width="200" height="100" x="10" y="10" rx="20" ry="20" fill="blue" />
</svg>
"""

png_bytes: list[bytes] = resvg_py.svg_to_bytes(svg_string=svg_string) # a large list of bytes
```

## Video

Here are the three most highly recommended, well-maintained, and well-packaged Python solutions for your specific workflow. 

Extracting frames, processing them, and re-assembling them while retaining the original audio requires a tool that understands both image arrays and audio/video multiplexing (muxing).

Here is the breakdown of the best tools for the job:

### 1. MoviePy
**Best for:** Ease of use and rapid development.

MoviePy is explicitly designed for this kind of high-level video editing. Under the hood, it uses FFmpeg to read/write video and audio, and NumPy to represent frames, making it incredibly easy to bridge with Pillow (PIL).

* **How the workflow looks:** You load the video into a `VideoFileClip`. You write a custom function that takes a NumPy array (the frame), converts it to a Pillow Image, processes it, and returns a NumPy array. You pass this function to `.fl_image()`. When you save the video, MoviePy automatically handles copying and syncing the original audio to the new file.
* **Pros:** Requires very few lines of code; audio retention is handled entirely automatically without needing to manually split or remux streams.
* **Cons:** Can be slower and more memory-intensive than lower-level libraries for very large or high-resolution videos.

### 2. PyAV (`av`)
**Best for:** High performance, robustness, and direct Pillow integration.

PyAV provides Pythonic bindings directly to FFmpeg's libraries (`libavformat`, `libavcodec`, etc.). It is heavily relied upon in the machine learning community (used by PyTorch's TorchVision) because of its speed and reliability.

* **How the workflow looks:** You open an input container and an output container. You iterate through the input packets. PyAV has a built-in `frame.to_image()` method that converts the video frame directly to a Pillow Image. After processing, you convert it back to a PyAV frame, encode it, and mux it into the output container. You also explicitly copy the audio packets from the input stream to the output stream.
* **Pros:** Blazing fast; offers exact, frame-accurate control over encoding and muxing; native Pillow support means no intermediate NumPy array conversion is strictly necessary.
* **Cons:** A steeper learning curve. The code is more verbose because you are manually managing streams, packets, and timing (PTS/DTS) for both video and audio.

### 3. OpenCV (`opencv-python`) paired with `ffmpeg-python`
**Best for:** Heavy computer vision processing and absolute control.

OpenCV is the undisputed industry standard for reading and manipulating image frames. However, OpenCV *cannot* handle audio. To solve your audio requirement, you pair it with `ffmpeg-python`, a fantastic wrapper that lets you run FFmpeg commands cleanly from Python.

* **How the workflow looks:** 1. Read frames sequentially using `cv2.VideoCapture()`.
    2. Convert OpenCV's default BGR format to RGB, pass it to Pillow, process it, convert it back, and write it to a temporary "video-only" MP4 using `cv2.VideoWriter()`.
    3. Use `ffmpeg-python` to combine your new video-only MP4 with the audio stream from the original MP4 into your final file.
* **Pros:** Unmatched speed for frame reading/writing; access to OpenCV's massive ecosystem of image processing tools if you need more than Pillow.
* **Cons:** Requires combining two distinct libraries and writing a temporary file to disk (or managing complex byte-pipes) to stitch the new video and original audio together.

---

**Summary Recommendation:**
If you want to get this done quickly with the least amount of boilerplate code, start with **MoviePy**. If you are building a production pipeline where processing speed, memory efficiency, and scale are your top priorities, use **PyAV**.
