
## .lines parser

Based on ./_private/lines2img.md and ./_private/lines2img.py and on the examples in ./_private/lines-examples/ and on the ACTUAL SOURCE CODE of the Vexy Lines app in ./_private/vexy-lines-src-readonly , write a complete parser for .lines files, and incorporate it into the vexy-lines-utils package.

## App-less export

Based on the .lines parser add to our package the functionality that’s in ./_private/lines2img.py (extraction of source and preview image). 

## GUI+CLI

Completely rework ./src/vexy_lines_utils/ so that: 

- We have an ability to get "style" = all effects from one .lines file and apply them to another .lines file. Ideally the proportions of the effects should be scaled to match the size of the target source image. For that, extensively use the .lines parser and the "MCP API" of Vexy Lines (see ./src/vexy_lines_utils/ and see for reference ./_private/mcp/ )
- Ultimately (though this can be deferred to later version), we want to be able to borrow style from 1st style .lines file and then also get a 2nd style .lines file, and try to match it to the 1st (= map a compatible structure of groups, layers, filters), and if the structure is the same but numeric parameters are different, then we want to interpolate the numeric parameters between the 1st style file and the 2nd style file. That means: this would be similar in principle to what we did in ./examples/mcp_create_video.py 
- Specifically, if we have 2 styles, then
  - if we have input images, we interpolate the style parameters between the 2 styles in as many steps as there are input images, and we apply each interpolated style to the corresponding input image to produce a sequence of output images or an output video
  - if we have an input video, we treat all the to-be-processed frames as input images, and we proceed as above
- The package must have a clean GUI based on ./_private/gui/style_with_vexy_lines.py incorporating ./_private/gui/ctkrangeslider.py directly and depending on the same packages that are used in ./_private/gui/style_with_vexy_lines.py 
- The outputs must support not just SVG, PNG and JPG, MP4 but also LINES format. 
- The conversion should be done like in ./examples/mcp_create_video.py, that is, we export to SVG using Vexy Lines MCP API, and then we convert to PNG using resvg, and then we convert to JPG or assemble into output video. 
- In addition to the GUI ported from ./_private/gui/style_with_vexy_lines.py, we want in the inputs section the 1st tab to be "Lines" (so there’s "Lines" | "Images" | "Video"). If "Lines" is selected, then the inputs section should look like the Images section, but the entire Style/End Style section should be disabled (grayed out). 
- The package must have a CLI that uses Fire CLI, and mirrors the GUI functionality (video-to-video, video-to-images, images-to-video, images-to-images with one or two style references). 
- The CLI should also support additional operations, like querying .lines files, extracting source and preview image from .lines files, simple batch export of .lines files to SVG or PNG or JPG or MP4. 
- In the GUI, if the Video tab area is active, dragdrop of a video to the entire Video tab area should work
- In the GUI, if I choose "Video > Add Video" from the menu, the inputs section should switch to the Video tab area
- In the GUI, If the Images tab area is active, dragdrop of multiple images to the entire Images tab area should work
- In the GUI, if I choose "Images > Add Images" from the menu, the inputs section should switch to the Images tab area
- In the GUI, If the Lines tab area is active, dragdrop of multiple .lines documents to the entire Lines tab area should work
- In the GUI, if I choose "File > Add Lines" from the menu, the inputs section should switch to the Lines tab area
- When we click the Export button, the app should produce 
    - a folder picker dialog to choose the output folder if the output format is SVG, PNG, JPG or LINES (the file naming should be automatic)
    - a file picker dialog to choose the output file if the output format is MP4
